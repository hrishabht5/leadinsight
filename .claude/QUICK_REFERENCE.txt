🎓 BEST PRACTICES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. ALWAYS START WITH /plan
     └─ Saves the most tokens (40-60%)
     └─ Prevents costly rewrites
     └─ Better solutions overall

  2. USE AGENT NAMES INSTEAD OF DESCRIPTIONS
     ✅ "Use Backend Architect from @agency-agents"
     ❌ "Design a scalable, modular architecture with separation of concerns..."

  3. FOLLOW TDD WORKFLOW
     └─ Write tests FIRST
     └─ Code comes after tests
     └─ Fewer bugs, faster implementation

  4. SIMPLIFY EVERYTHING
     └─ /simplify after every coding session
     └─ Remove unnecessary complexity
     └─ Improves code quality

  5. CHAIN EXPERTS TOGETHER
     └─ Backend Architect + Security Specialist + Code Reviewer
     └─ Professional-grade output
     └─ Catches issues early
